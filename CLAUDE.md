# Clone App — Project Rules

## 참조 사이트

### 원본 (UI/UX 기준)
- **https://bubblealba.com/** — 이 앱의 모방 대상. UI/UX/기능 구현 시 원본을 기준으로 판단.
- 기능 추가/수정 요청이 애매하면 원본 사이트 동작을 참고하여 결정.
- 원본과 다르게 동작하는 부분 발견 시 사용자에게 알릴 것.

### 기능 참조 (UI 변경 없이 기능만 참고)
- **https://adultjob2.cgimall.co.kr/** — CGImall 성인구인구직 솔루션. 기능 추가 시 참조.
- **https://adultjob2.cgimall.co.kr/admin/** — CGImall 관리자 패널. 관리자 기능 확장 시 참조.

- **원칙**: UI는 bubblealba 스타일 유지, 기능 로직만 cgimall 참조하여 이식.

## UI 변경 금지
- **현재 UI를 절대 변경하지 않는다.** 레이아웃, 색상, 폰트, 간격, 컴포넌트 구조 모두 유지.
- 기능 추가/수정 시 기존 UI 안에서만 동작하도록 구현.
- 새 페이지/컴포넌트 추가가 불가피하면 기존 디자인 패턴을 그대로 따를 것.

## 개발 도구
- **Prisma Studio** (`localhost:5556`) — DB 테이블/데이터 실시간 확인용. 기능 개발·디버깅 시 실제 데이터 구조와 값을 확인하고 작업할 것.
- 실행: `npx prisma studio --port 5556`

## 수정 후 필수 검증
코드 수정 후 **반드시** 아래 체크리스트를 실행:

1. **타입체크**: `npx tsc --noEmit` — 에러 0개 확인
2. **페이지 오류 확인**: 수정한 파일이 영향을 주는 페이지를 `npm run dev`로 실행 후 브라우저에서 직접 확인
   - 페이지가 정상 렌더링되는지 (흰 화면, 500 에러 없는지)
   - 콘솔에 런타임 에러가 없는지
   - 수정한 기능이 의도대로 동작하는지
3. **관련 페이지 연쇄 확인**: 공통 컴포넌트/라이브러리 수정 시 해당 컴포넌트를 사용하는 **다른 페이지도** 깨지지 않았는지 확인
4. **증거 없이 "완료" 선언 금지** — tsc 통과 + 페이지 정상 확인 스크린샷/로그 필수
